# FCE003: NonLiteralErrorCode

🌍 **Langues:**  
🇬🇧 [English](./FCE003.en.md) | 🇫🇷 Français (ce fichier)

| | |
|---|---|
| **Catégorie** | Codes d'erreur (`FirstClassErrors.ErrorCodes`) |
| **Sévérité** | 🔵 Info |
| **Activée par défaut** | Non (opt-in) |

`ErrorCode.Create` est appelé avec un argument qui n'est pas une constante de compilation. Un tel code est un angle mort pour la détection de doublons ([FCE001](FCE001.fr.md)) et les vérifications de format. Activez cette règle si votre équipe veut que chaque code d'erreur reste littéral.

## Non conforme

```csharp
private static string Build() => "DYNAMIC";
public static readonly ErrorCode A = ErrorCode.Create(Build()); // FCE003
```

## Conforme

```csharp
public static readonly ErrorCode A = ErrorCode.Create("MONEY_TRANSFER_INVALID");
```

## Activer cette règle

Cette règle est désactivée par défaut. Activez-la dans `.editorconfig` :

```ini
dotnet_diagnostic.FCE003.severity = suggestion   # ou warning / error
```
