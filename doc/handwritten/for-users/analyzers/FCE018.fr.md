# FCE018: OversizedErrorContextValue

🌍 **Langues:**  
🇬🇧 [English](./FCE018.en.md) | 🇫🇷 Français (ce fichier)

| | |
|---|---|
| **Catégorie** | Usage (`FirstClassErrors.Usage`) |
| **Sévérité** | 🔵 Info |
| **Activée par défaut** | Non (opt-in) |

Une `ErrorContextKey` est déclarée avec un type de valeur qui est un gros payload : un tableau d'octets, un `System.IO.Stream` (ou un sous-type), ou un `System.IO.FileInfo`. Le contexte d'erreur est fait pour de petits faits faciles à logger ; un fichier ou un buffer entier alourdit chaque ligne de log et chaque entrée du catalogue d'erreurs, et y fait souvent passer des données sensibles.

La détection repose sur le **type de valeur déclaré** de la clé : elle se déclenche une fois, là où la clé est créée.

## Non conforme

```csharp
public static readonly ErrorContextKey<byte[]> Attachment =
    ErrorContextKey.Create<byte[]>("ATTACHMENT"); // FCE018

public static readonly ErrorContextKey<System.IO.Stream> Body =
    ErrorContextKey.Create<System.IO.Stream>("REQUEST_BODY"); // FCE018
```

## Conforme

Stockez un petit descripteur stable du payload plutôt que le payload lui-même :

```csharp
// Identifiez le fichier ; ne transportez pas ses octets.
public static readonly ErrorContextKey<string> FileName =
    ErrorContextKey.Create<string>("FILE_NAME");

public static readonly ErrorContextKey<long> FileSizeBytes =
    ErrorContextKey.Create<long>("FILE_SIZE_BYTES");
```

Voir [Contexte d'erreur — Quand ne pas l'utiliser](../ErrorContext.fr.md#-quand-ne-pas-lutiliser).

## Activer cette règle

Cette règle est désactivée par défaut. Activez-la dans `.editorconfig` :

```ini
dotnet_diagnostic.FCE018.severity = suggestion   # ou warning / error
```
