# FCE018: OversizedErrorContextValue

🌍 **Languages:**  
🇬🇧 English (this file) | 🇫🇷 [Français](./FCE018.fr.md)

| | |
|---|---|
| **Category** | Usage (`FirstClassErrors.Usage`) |
| **Severity** | 🔵 Info |
| **Enabled by default** | No (opt-in) |

An `ErrorContextKey` is declared with a value type that is a bulk payload: a byte array, a `System.IO.Stream` (or subtype), or a `System.IO.FileInfo`. Error context is meant for small, loggable facts; a whole file or buffer bloats every log line and error-catalog entry, and often smuggles sensitive data along with it.

Detection is based on the key's **declared value type**, so it fires once where the key is created.

## Noncompliant

```csharp
public static readonly ErrorContextKey<byte[]> Attachment =
    ErrorContextKey.Create<byte[]>("ATTACHMENT"); // FCE018

public static readonly ErrorContextKey<System.IO.Stream> Body =
    ErrorContextKey.Create<System.IO.Stream>("REQUEST_BODY"); // FCE018
```

## Compliant

Store a small, stable descriptor of the payload instead of the payload itself:

```csharp
// Identify the file; do not carry its bytes.
public static readonly ErrorContextKey<string> FileName =
    ErrorContextKey.Create<string>("FILE_NAME");

public static readonly ErrorContextKey<long> FileSizeBytes =
    ErrorContextKey.Create<long>("FILE_SIZE_BYTES");
```

See [Error Context — When not to use it](../ErrorContext.en.md#-when-not-to-use-it).

## Enabling this rule

This rule is disabled by default. Turn it on in `.editorconfig`:

```ini
dotnet_diagnostic.FCE018.severity = suggestion   # or warning / error
```
