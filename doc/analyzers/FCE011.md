# FCE011: DuplicateDocumentedCode

| | |
|---|---|
| **Category** | Documentation content (`FirstClassErrors.DocumentationContent`) |
| **Severity** | 🔴 Error |
| **Enabled by default** | Yes |

More than one documented factory produces the same error code by referencing the same `ErrorCode` field. Documentation extraction groups by code and keeps a single entry, so the others collapse silently. This complements [FCE001](FCE001.md), which only sees duplicate `ErrorCode.Create` literals (a shared field has just one).

## Noncompliant

```csharp
[DocumentedBy(nameof(DocA))]
internal static DomainError A() => DomainError.Create(Code.Shared, "...").WithPublicMessage("...");

[DocumentedBy(nameof(DocB))]
internal static DomainError B() => DomainError.Create(Code.Shared, "...").WithPublicMessage("..."); // FCE011: same code
```

## Compliant

```csharp
// A() uses Code.A, B() uses Code.B — one documented error per code.
```

## Details

This is a compilation-end diagnostic: it appears at build / full-solution analysis.
